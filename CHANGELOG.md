# 變更紀錄 (Changelog)

本專案版本格式遵循 [Semantic Versioning](https://semver.org/lang/zh-TW/)。

## [未發布]

### 新增
- **排除清單可從開啟中程式挑選**：設定視窗的「排除的執行檔」上方新增下拉選單，列出目前開啟中視窗的執行檔名（去重、排序），選擇後按「加入」即附加到排除清單；另有「重新整理」可重新掃描。免手動輸入路徑或檔名。

## [1.1.0] - 2026-06-16

### 新增
- **螢幕配置變更時自動還原**：第五種觸發機制，監聽螢幕配置變更（插拔螢幕 / 改解析度 / 排列變動），去抖後比對螢幕配置簽章，確實改變才自動還原佈局。預設關閉、可獨立開關、設定持久化。靠顯示設定事件觸發並重用還原冷卻機制，與「新視窗自動還原」同開也不會形成迴圈。

### 變更
- **還原時依視窗現況保留最大化/最小化狀態**：還原不再依記錄狀態強制改變視窗——目前最大化的維持最大化、最小化的維持最小化，僅一般視窗會直接套用記錄的位置/大小（三種情況都會更新還原後的一般位置矩形）。

### 移除
- 移除「還原時保留最小化狀態」設定：在上述新還原邏輯下已無作用。舊 `settings.json` 殘留該欄位會被自動忽略，無相容問題。

## [1.0.1] - 2026-06-16

### 修正
- **修正 Windows 11 上「新視窗自動還原」的無限迴圈**：還原視窗時 `SetWindowPlacement` 會讓目標視窗再發出 `EVENT_OBJECT_SHOW`，於 Win11（對顯示事件特別敏感）造成「還原 → 觸發事件 → 再還原」不斷重觸發。新增還原冷卻機制（剛還原過的視窗 3 秒內不再回應其顯示事件），手動「全部還原」搬動的視窗也納入冷卻，避免連鎖重觸發。

## [1.0.0] - 2026-06-15

第一版。Windows 用的常駐視窗管理工具，記憶並還原應用程式視窗位置。

### 新增
- **視窗辨識**：以執行檔路徑 + 視窗類別名為主鍵、視窗標題模糊比對為加權、螢幕配置簽章一致加分；可設定比對門檻與排除清單。
- **累加記憶佈局**：以「執行檔路徑 + 視窗類別名 + 視窗標題」為鍵 upsert 合併，保留目前未開啟的視窗記錄；可設保留上限（預設 500，超過淘汰最舊）與清除所有記錄。
- **持久化**：佈局/設定以 JSON 存於 `%AppData%\WindowManager\`，原子寫入避免毀損，含版本與毀損降級處理。
- **多螢幕安全還原**：座標離屏時夾限至最接近的可見螢幕工作區。
- **四種觸發機制（皆可獨立開關）**：全域快捷鍵手動儲存/還原（可自訂）、自動定時儲存、新視窗自動還原（WinEventHook）、開機/登入後自動還原。
- **系統列常駐 + 設定 UI**：右鍵選單、設定視窗、開機自動啟動（HKCU Run）。
- **命令列模式**：`--save` / `--restore` / `--clear`。

### 預設值
- 快捷鍵：儲存 `Ctrl+Alt+S`、還原 `Ctrl+Alt+D`（避開常被佔用的 `Ctrl+Alt+R`）。
- 自動定時儲存、新視窗自動還原、開機後自動還原預設關閉。

### 測試
- 33 個單元測試（xUnit）涵蓋比對評分、持久化降級、離屏夾限、累加合併。
- 實機驗證：快捷鍵儲存/還原、新視窗自動還原、累加記憶（真實視窗）。

### 已知限制
- 無法操作以系統管理員權限執行的視窗（自動略過）。
- 不支援 UWP/沙箱化視窗、虛擬桌面佈局。
- 重開機/登入後自動還原尚未實機驗證（需實際重開機）。

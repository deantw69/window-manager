## Why

在 Windows 上，應用程式視窗的位置與大小常在「關閉後重開」或「重新開機 / 登入」後跑掉，多螢幕環境下尤其嚴重（換螢幕、解析度變動會把視窗丟到看不見的座標）。本專案要做一個常駐的視窗管理工具，記憶每個應用程式視窗的位置並在指定時機自動還原，免去每次手動排版。

## What Changes

- 新增一個 Windows 常駐工具（C# / .NET，WPF + 系統列圖示），透過 Win32 API 擷取與設定視窗位置/大小。
- 以「執行檔路徑 + 視窗標題（含模糊比對）+ 視窗類別名 + 當時螢幕配置」綜合辨識同一個應用程式視窗。
- 將視窗佈局持久化到本機檔案，重開機後仍可讀取還原。
- 提供四種觸發時機，且每一項都可由使用者開關 / 設定：
  - 快捷鍵手動儲存 / 還原（快捷鍵可自訂）。
  - 自動定時儲存（可開關、可設定間隔）。
  - 偵測到符合的新視窗時自動還原（可開關）。
  - 開機 / 登入後自動還原（可開關）。
- 提供設定 UI（系統列選單 + 設定視窗）管理上述開關、快捷鍵與規則。
- 多螢幕安全還原：螢幕配置改變時避免把視窗放到不存在的座標。

## Capabilities

### New Capabilities
- `window-identity`: 定義如何辨識「同一個應用程式視窗」——以執行檔路徑、視窗標題（模糊比對）、視窗類別名為依據，並記錄當時的螢幕配置；以及多筆候選的比對與評分規則。
- `layout-persistence`: 擷取視窗位置/大小並持久化到本機檔案，定義資料結構、版本、與重開機後的讀取行為。
- `position-restore`: 將已儲存的佈局套用到目標視窗（含多螢幕安全檢查、座標夾限），定義還原的成功/失敗與略過條件。
- `restore-triggers`: 四種儲存/還原觸發機制（快捷鍵、定時自動儲存、新視窗自動還原、開機登入後還原），每項皆可獨立開關，快捷鍵可自訂。
- `tray-app`: 系統列常駐外殼與設定 UI——啟動、選單、設定視窗、開機自動啟動、各項開關與快捷鍵設定的管理。

## Impact

- 全新專案，無既有程式碼受影響。
- 技術依賴：.NET（WPF）、Win32 API（user32.dll：`EnumWindows`、`GetWindowRect`、`SetWindowPos`、`GetWindowThreadProcessId` 等）、全域熱鍵註冊（`RegisterHotKey`）、多螢幕資訊（`EnumDisplayMonitors` 或 .NET `Screen`）。
- 持久化：本機設定/佈局檔案（如 `%AppData%` 下的 JSON）。
- 系統整合：開機自動啟動（登錄機碼或啟動資料夾）。
